# Skill: docker-type-check

Sets up two TypeScript type-checking gates for any Next.js / TypeScript project running in Docker:

1. **Pre-commit hook** — runs `tsc --noEmit` inside the running dev container on every `git commit`. Blocks the commit if types fail. Soft-fails (warns, allows commit) when Docker isn't running.
2. **Dockerfile build gate** — adds `RUN pnpm run type-check` (or `npm run type-check`) before `RUN ... build` in the builder stage. Catches type errors in CI/CD before the full webpack build runs.

## When to use

- Project uses TypeScript + Docker for local dev
- The dev server skips type-checking (Turbopack, Vite, `ts-node`, etc.)
- Production builds fail on type errors that weren't caught locally
- Works with: Next.js, Vite, Create React App, NestJS, any project with a `type-check` npm script

## Quick Start

```
/setup-docker-type-check
```

The agent will detect your project layout and ask for:
- Docker container name (e.g. `myapp-frontend-dev`)
- Package manager (`pnpm` / `npm` / `yarn`)
- Whether to patch the Dockerfile as well

## Steps the agent performs

### 1. Detect project layout

```bash
# Find package.json with type-check script
grep -r '"type-check"' . --include="package.json" -l

# Find existing Dockerfile(s)
find . -name "Dockerfile*" -not -path "*/node_modules/*"

# Check for existing .githooks/
ls .githooks/ 2>/dev/null
```

### 2. Verify / create type-check script in package.json

Checks that `type-check` script exists. If missing, adds:

```json
"type-check": "tsc --project tsconfig.json --noEmit"
```

If `tsconfig.typecheck.json` exists, uses that instead (faster — excludes test files):
```json
"type-check": "tsc --project tsconfig.typecheck.json --noEmit"
```

### 3. Create `.githooks/pre-commit`

Creates `.githooks/` if it doesn't exist, writes the hook:

```bash
#!/bin/bash
# Pre-commit type-check hook
# Generated by docker-type-check skill
# Container: {{CONTAINER_NAME}}
# Package manager: {{PKG_MANAGER}}

echo "🔍 Running pre-commit type check..."

# Detect Docker (handles WSL, Linux, macOS)
DOCKER_CMD=""
if command -v docker &>/dev/null && docker info &>/dev/null 2>&1; then
  DOCKER_CMD="docker"
fi

if [ -z "$DOCKER_CMD" ]; then
  echo "⚠️  Docker not running — skipping type check (run manually: docker exec {{CONTAINER_NAME}} {{PKG_MANAGER}} run type-check)"
  exit 0
fi

echo "   Checking TypeScript types in container {{CONTAINER_NAME}}..."
$DOCKER_CMD exec {{CONTAINER_NAME}} {{PKG_MANAGER}} run type-check

if [ $? -ne 0 ]; then
  echo ""
  echo "❌ TypeScript type check failed. Fix errors above before committing."
  echo "   Run manually: docker exec {{CONTAINER_NAME}} {{PKG_MANAGER}} run type-check"
  exit 1
fi

echo "✅ Type check passed!"
exit 0
```

Makes it executable: `chmod +x .githooks/pre-commit`

### 4. Activate the hook

```bash
git config core.hooksPath .githooks
```

Verify activation:
```bash
git config core.hooksPath
# Should output: .githooks
```

### 5. Patch Dockerfile (optional)

If the user confirms, finds the builder stage in the Dockerfile and inserts `RUN {{PKG_MANAGER}} run type-check` before the build step.

**Before:**
```dockerfile
RUN --mount=type=cache,target=/app/.next/cache \
    pnpm run build
```

**After:**
```dockerfile
# Type-check before building — fails fast with clear tsc errors
RUN pnpm run type-check

RUN --mount=type=cache,target=/app/.next/cache \
    pnpm run build
```

Pattern to find: `pnpm run build` or `npm run build` line in a `RUN` instruction.

### 6. Update CLAUDE.md / project docs

Appends a note to the project's CLAUDE.md under the Testing Workflow section:

```markdown
**Pre-commit type check:** Active via `.githooks/pre-commit`.
Runs `{{PKG_MANAGER}} run type-check` in Docker container `{{CONTAINER_NAME}}` on every commit.
Activate with: `git config core.hooksPath .githooks`
```

### 7. Verify end-to-end

```bash
# Confirm hook is active
git config core.hooksPath

# Confirm container is running and type-check works
docker exec {{CONTAINER_NAME}} {{PKG_MANAGER}} run type-check

# Test the hook fires on commit (optional)
# Introduce a deliberate type error, attempt commit, confirm it's blocked
```

## Notes

- **Soft-fail design:** The pre-commit hook warns but does NOT block when Docker isn't running. This avoids breaking commits on machines without Docker (e.g. CI systems that don't use Docker for the hook). The Dockerfile gate is the hard fail for CI/CD.
- **Container must be running:** The hook calls `docker exec` — the dev container must be up. If it's not, the hook soft-fails.
- **`tsconfig.typecheck.json` vs `tsconfig.json`:** If the project has a separate typecheck config (excludes tests, node_modules), prefer it — it's faster. Otherwise fall back to the main tsconfig.
- **Windows (Git Bash / WSL):** The hook uses `command -v docker` which works in both Git Bash and WSL. On Windows without WSL, `docker info` may require Docker Desktop to be running.
